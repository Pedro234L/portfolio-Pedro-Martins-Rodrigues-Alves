# 📱 Qr-Transformer: Engenharia Reversa, Acessibilidade e Firebase
 ## 📝 Descrição do Projeto
Este projeto explora a **Engenharia Reversa Assistida por IA**, focando na reconstrução lógica e funcional de sistemas sem o acesso direto ao código-fonte original. O objetivo foi transformar requisitos descritivos em um software funcional, integrando diferenciais de acessibilidade e uma infraestrutura robusta em nuvem.

O desenvolvimento serviu como base para uma análise crítica sobre a formação de engenheiros na era da IA, os limites éticos da originalidade digital e a implementação de serviços modernos de Backend-as-a-Service (BaaS).

 ## 🛠️ Integração com Firebase
Para garantir persistência e segurança, o projeto utiliza os seguintes recursos do **Firebase**:

* **Authentication:** Sistema de login para que cada usuário possa salvar e gerenciar suas preferências de QR Code com segurança.
* **Cloud Firestore:** Banco de dados NoSQL utilizado para armazenar o histórico de ações e as configurações personalizadas de cada modelo criado.
* **Cloud Storage:** Armazenamento seguro de arquivos e ativos gerados pela ferramenta.

 ## 🚀 Diferenciais de Acessibilidade
O projeto destaca-se pela inclusão de recursos que garantem uma experiência inclusiva:

1. **Vlibras:** Navegação assistida para pessoas com deficiência auditiva.
2. **Suporte para Daltônicos:** Ajuste de paleta de cores do QR Code conforme o tipo de daltonismo do usuário.
3. **Modelos e Comunidade:** Aba dedicada para compartilhamento de templates entre usuários.
4. **Navegação via Teclado:** Suporte completo para uso do sistema sem a necessidade de mouse.

 ## 📊 Reflexão Crítica: O Desenvolvedor e a IA
Com base na experiência de engenharia reversa, foram consolidadas as seguintes análises:

* **A Formação na Era da IA:** A mudança da escrita sintática para a descrição lógica exige que o engenheiro júnior domine a **Engenharia de Prompt** e mantenha o **Pensamento Crítico** para não se tornar obsoleto.
* **Originalidade vs. Plágio:** A prática deixa de ser aprendizado quando não há participação direta na construção da lógica.
* **Política de Explicação:** Propõe-se que o desenvolvedor documente sua jornada de criação e comandos utilizados como diretriz ética para comprovar o valor agregado ao código gerado.
