# 🤖 Engenharia de Software Assistida por IA
 ## 📝 Descrição do Projeto
Este projeto explora o impacto da Inteligência Artificial no desenvolvimento de software moderno, focando na integração entre a automação de código e o pensamento crítico do desenvolvedor. O objetivo é analisar como ferramentas de IA podem acelerar a produção sem comprometer a integridade técnica e a originalidade do profissional.

O sistema demonstra a aplicação de prompts estruturados para a criação de soluções funcionais, servindo como base para o estudo da autonomia técnica em um cenário onde a IA atua como co-piloto no processo de engenharia.

 ## 📊 Reflexão Crítica e Aprendizados
Durante o desenvolvimento, foram analisados pontos cruciais sobre o futuro da profissão de Engenharia de Software:

* **Competências Essenciais:** Identificamos que, para evitar a dependência excessiva da IA, o engenheiro junior deve focar na **Autonomia** e na **Capacidade de criar softwares funcionais** por conta própria. A técnica deve ser uma ferramenta, não uma muleta.
* **Originalidade vs. Plágio:** O desenvolvimento passa a ser considerado plágio quando a maioria dos elementos do software não é pensada pelo desenvolvedor.
* **Valor Agregado:** Como medida mitigadora, o profissional deve comprovar que adicionou valor real ao software, indo além da simples cópia de saídas geradas por IA, garantindo que a lógica e a arquitetura foram compreendidas e validadas.
